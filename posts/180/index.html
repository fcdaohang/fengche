前言#

荣幸地向大家介绍-Analog–它是世界上第一个由时间证明提供支持的全链网络。 Analog 是目前基于区块链的加密网络新生代之一，它承诺提供一组和同类项目区别明显的漂亮功能。
究竟是什么让它如此特别？答案与 Web 3.0 未来中事件数据的利用有关。

重新思考事件数据#

我们的世界充满了各种具有相应数据的事件。
无论是数字交易、在线预订、天气更新还是社交媒体帖子，全球无数来源每分每秒都会生成事件数据。

由于事物位于当前的 Web 2.0 环境中，因此这些数据中的大部分都是在独立的平台（例如Twitter）中独立保存和使用的。尽管跨平台事件数据通信可能有很大的好处，但目前这样做的尝试通常是一片混乱，充满了低效率和不兼容的接口。

这就是 Analog 的激进计划完美契合即将到来的 Web 3.0 之处——构建一个全链网络，为分散而基于时间的应用程序提供构建模块。该网络将使事件数据能够在需要的时间和地点以安全的方式跨链和 dApp 调用，且不会受到因第 1 层区块链的可扩展性问题造成的阻碍。

Analog 团队是这么解释的：“Analog 正在创建一个系统来验证事件数据，该系统将继续成为我们所谓的 Event Data Marketplace（事件数据市场） 的一部分，它可以在 Analog 的 Timechain 上的 dApp 中的智能合约中使用。简单来说，Analog 想要解放区块链中原本隐蔽的事件数据，并通过去中心化的方式对其进行访问，使其能够充分发挥其潜力。”

Analog 的产品至关重要。除了其改变游戏规则的事件数据驱动的生态系统外，该项目实现高效全链互操作性的方法将在 Web 3.0 未来发挥决定性作用。
当前的第 1 层区块链互操作性解决方案经常缺乏，例如在 DeFi 领域中可以看到——由于不同平台之间的多链桥的限制，在不同的链之间转移流动性并不容易。

它如何运作#

要了解 Analog 的区块链网络如何运作，您需要熟悉它的一些基础知识；即 Timechain（时间链）、Timegraph（时间图）、Event Data Marketplace （事件数据市场）和 nodes（节点）。

Timechain 本质上是 Analog 的账本，配备了可扩展和适应的时间证明（Proof-of-Time）共识机制，用于建立无可争议的事件历史。
Analog 团队是这样描述的：“这是世界上第一个去信任的全链互操作性协议，它提供了一个强大的、低级的通信原语，开发人员可以在该原语上实现跨链应用程序。使用这个新的原语，Analog 网络为下一代 dApp 的开发铺平了道路。”

事件数据被馈送到时间链并在可用于事件数据市场之前进行验证。单个节点的事件数据提交累积到 时间图中。 时间图 API 用于跨链和 dApp 安全地传输事件数据。
事件数据市场的功能可理解为用于验证事件数据的去中心化店铺，这些数据可以由节点以无需信任的点对点 (P2P) 方式进行交换。

有一些节点类型与时间链交互。有时间节点（确认区块）、时间选举者（提议区块）、发布者（提交区块）和Tesseract 节点。 Tesseract 节点与其他节点有点不同，它不仅从外部链检索事件数据，还维护和运行跨链路由以及事件数据从网络到目标链的传输。

事件数据通过启用隐私的递归零知识证明协议进行保护。这种安全措施允许在应用程序之间跨链和双向传输事件数据，而不会损害底层事件数据本身。

PoT时间证明的优势#

为时间链专门创建的时间证明 (PoT) 算法在其运行方式上是独一无二的。
它不是像工作量证明设置那样衡量计算工作量，也不是像权益证明系统那样检查验证者持有的股份数量，而是利用可验证的延迟函数（VDF）加上排名分数和质押代币来伪随机选择时间选举者（区块提议者）和时间节点（区块确认者）。

任何节点都可以成为时间选举者并提出新块，前提是它具有更高的排名分数——根据节点的事件相关性、声誉和相邻节点的平均加权值计算——以及最小锁定代币。 PoT 还对恶意攻击具有很强的抵抗力，因为每个黑客都需要超过 99% 的排名分数才能操纵网络。

多功能网络#

Analog 在可互操作的全链环境中强大而有效地使用事件数据为 dApp 提供动力，使其成为跨多个行业众多用例的理想选择。

例如，个人在虚拟平台上奖励 NFT，就像 Decentraland。由于 Analog 采用全链配置，该平台能够在其他区块链平台（如 Sandbox 和 OpenSea）上触发操作，比如接收 NFT 的确认时间——这些平台原本是孤立的。

在另一个涉及 DeFi 的用例示例中，画面转向 John Doe 先生。他想要执行将在多个区块链平台上进行的交易。
由于多链桥的互操作能力有限，这将是一件麻烦事，其特点是 Doe 先生必须忍受一系列繁琐流程，例如跨链交换和转换原生代币及其相关的麻烦。在这堆操作中潜伏的人为错误的风险也更高。

随着 Analog 的第 0 层生态系统提高了全链级别的互操作性，由于所涉及的链之间几乎无缝连接，类似的交易将变得不那么乏味。这意味着由交易触发的事件数据将在涉及的不同区块链平台之间自主且安全地共享。 Doe 先生无需担心烦人的中介过程。

开发者的梦想#

凭借无限的潜在用例，Analog 为在 Timechain 上构建同样多产的 dApp 打开了大门。
随着项目的进展，Analog 欢迎开发人员在测试网上扩展他们的编码技能，Analog 会提供详细的 SDK、API 文档和 Analog Continuum 智能合约脚本系统。

届时利用事件数据市场上可用的分散事件数据构建基于事件的第三方应用程序。这消除了依赖将控制权置于单一实体上集中式服务的需要。

TIME 代币#

Analog 使用原生 $ANLOG 代币激励其网络，该代币用于事件数据交易并奖励验证者（时间节点）在网络上的积极行动。通过购买和持有代币，Analog 社区也拥有该项目的一部分。

截至 2022 年第一季度，Analog 正在努力完成涉及机构和战略投资者的种子轮融资。该公司还在与一些去中心化和集中式交易所（DEX 和 CEX）就在不久的将来上市 $ANLOG 进行谈判。

2021–2022 路线图#

Analog 于 2021 年第三季度开始了它的旅程，目前正以充满希望的速度前进，并在此过程中已抵达关键的里程碑。
它获得了区块链和加密社区的大力支持，其潜力得到了行业重量级人物的认可，例如 Binance 的 BNB Chain，他将该项目纳入其梦寐以求的 2022 年第一季度 MVB IV 孵化计划——该计划有助于推动区块链技术的发展创业公司。
