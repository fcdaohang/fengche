IOTA 基金会和 walt.id 为世界带来无缝登录

长文预警#

IOTA 基金会和 walt.id 正在开发一种保护隐私、基于 SSI 的登录系统，允许传统 (Web2) 和去中心化 (Web3) 应用程序轻松登录用户。用户仍然可以完全控制他们的数据，并且能够自行决定安全地共享信息。

IOTA 基金会正在开发一种基于自我主权身份 (SSI) 的隐私保护登录系统，称为 Login With IOTA，它允许传统 (Web2) 和去中心化 (Web3) 应用程序轻松登录用户。在这个系统中，用户可以完全控制他们的数据，并且能够自行决定安全地共享信息。

今年 3 月，我们发起了一项提案请求，以寻找合作伙伴来帮助实施 Login With IOTA。我们感谢所有参与者提交的提案：坦率地说，我们被所有人的关心和想法所震撼，最终选择了 walt.id 的提案，该提案涵盖了 Web2 和 Web3 解决方案。

Walt.id 是为开发人员和企业提供身份、NFT 和钱包基础设施的提供商，提供从概念、实施到托管云服务支持的专业知识和服务。该公司的产品都是开源的，被政府、公共机构、企业和去中心化自治组织用于快速构建 Web3 身份、NFT 和钱包的应用程序和用例，并且没有太多复杂性。

提议的解决方案将提供保护隐私的登录系统，该系统可以轻松地与任何应用程序集成，并遵循明确定义的标准以实现最大的互操作性。

由于与 walt.id 的合作，IOTA Identity 将被集成到 walt.id 的开源 SSI 基础设施解决方案中，这将能够支持 IOTA 生态系统以及其他身份生态系统，如 EBSI（欧盟区块链服务基础设施）， Gaia-X、Velocity Network、以太坊等。

什么是使用 IOTA 登录？#

Login With IOTA 允许网站和应用程序在尊重用户隐私、减少摩擦并提供增强的安全性的同时吸引用户。

该系统将与 OpenID Connect (OIDC) 等现有标准互操作，超过 50,000 个网站使用超过 10 亿个帐户。不幸的是，这些现有帐户主要集中在大型技术提供商周围，其数据商业化损害了用户的隐私。

在之前的博客文章中，我们讨论了为什么这种集中化存在问题以及为什么我们需要使用 IOTA 登录。如果您还没有，请随时返回并阅读博客文章！在这篇博文中，我们将更详细地描述提议的解决方案，并了解这些解决方案将支持什么样的用例。

项目目标#

Login With IOTA 的主要目标是开发一个登录系统，该系统可以利用去中心化 SSI 等尖端数字身份方法，并弥合这些方法与当今用于单点登录 (SSO) 的身份基础设施之间的差距。因此，这个项目有两个部分：

  • Web2 SSO：这将实现新的去中心化身份范式 (SSI) 与传统的集中式身份和访问管理 (IAM) 基础设施和工具（例如 KeyCloak）之间的向后兼容性。这部分对于构建和维护“传统”Web2 应用程序的开发人员和企业来说将是有价值的。

  • Web3 SSO：这将通过直接利用 IOTA 身份框架提供基于 SSI 的用户入职流程。这部分对于正在构建 Web3 dApp 的开发人员和企业来说将是有价值的。

为了为快速和广泛采用铺平道路，该项目将使用 OIDC 等全球公认的标准进行身份验证和数据交换，并与广泛使用的 W3C 可验证凭证对身份数据进行建模。

由于数据安全是数字身份不可或缺的一部分，因此数据主权、数据保护和隐私对于该项目的采用至关重要。这就是为什么这些领域将在解决方案的设计和实施中得到特别考虑。例如，身份提供者将被实现为盲人参与者，它几乎不记录和存储信息。

开发人员的经验也很重要，包括开放性（在标准和接口方面）、服务可组合性、互操作性、复杂性抽象以及与现有基础设施集成的便利性、开发和测试工具以及文档等因素。因此，Login With IOTA 将实现轻松采用并显着缩短实施的上市时间：添加新的身份提供者将成为几分钟的事情，因为该解决方案遵循明确定义的标准并且可以与任何兼容的库一起使用。

它是如何工作的？#

我们设想的 walt.id 系统将允许您有选择地共享个人数据，例如您的电子邮件地址以及您通常必须重复提供的任何其他信息，例如您的地址、电话号码、出生日期等。不同之处在于，您只需在您的 SSI 钱包中输入一次此信息，当您访问支持该技术的网站时，可以创建更流畅的体验，就像使用集中式提供商一样。但在这种情况下，您仍然可以完全控制数据。此外，这还允许您使用 Verifiable Credentials 机制共享可验证信息。

使用 Verifiable Credentials，您不仅可以共享信息，还可以证明信息的有效性。您共享的信息必须由受信任的第三方签署，例如政府机构、银行或其他可信赖的实体，但您可以决定与谁共享此信息。请求者有责任证明他们需要您提供这些信息并解释您将获得什么回报。

简而言之，实施的结果将是两种解决方案，一种针对现有生态系统，我们称之为 Web2 SSO，另一种针对前瞻性标准，我们称之为 Web3 SSO。

Web2 解决方案#

Web2 解决方案将完全符合广泛采用的 OIDC 标准，允许通过简单的配置更改采用应用程序来载入身份。该项目的一个成果将是提供一个身份提供者，作为 SSI 和传统 Web2 世界之间的桥梁。该组件可以由任何人托管，允许多个值得信赖的提供商并存。这很重要，因为 Web2 登录仍然需要身份提供者在一个稳定的域下可访问，该域最终必须由某人控制。因此，选择您信任的人将您的 SSI 信息传递给传统应用程序仍然很重要。

开发的解决方案将与传统的中心化身份提供者一起工作，因此应用程序开发人员可以混合和匹配身份提供者，从而根据需要混合和匹配身份生态系统。

从更技术的角度来看，所提出的解决方案将按如下方式工作。一旦钱包用户想要使用他们的身份凭证登录服务（例如，他们单击“连接钱包”按钮），现有的身份和访问管理解决方案将通过称为“联合”的过程将用户转发给身份提供者。然后，身份提供者与用户的钱包连接并请求身份凭证，然后可以根据可定制的策略对其进行验证。验证结果被发送回身份提供者并转换为传统 IAM 工具可以使用的格式（例如 JSON Web Token）。

虽然 IDP 工具包将能够与现有的 OIDC IAM 解决方案集成，但它也可以直接用于可能不需要成熟的 IAM 工具的更简单的用例。从用户的角度来看，体验将完全相同。

Web2 方法对于已经集成了 OIDC 并希望提供更多隐私保护选项的现有应用程序非常有用。

Web3 解决方案#

Web3 解决方案是围绕一个名为 Self-Issued OpenID Connect Provider (SIOP) 的新兴标准构建的，它允许用户以完全保护隐私的方式注册和登录。这种方法不需要集中的身份提供者，因为每个身份钱包或代理都充当自己的提供者。这允许用户和应用程序在利用自我主权身份的同时建立对等身份验证。

此外，可以通过此经过身份验证的通道共享声明或凭据，从而允许在身份验证范围之外进行丰富的交互。例如，可以共享提供年龄证明的凭证以购买有年龄限制的商品和服务。

Web3 方法对于希望从一开始就获得隐私的新建应用程序或现有应用程序过渡到的应用程序将非常有趣。

如果您希望我们及时了解该项目的进展和启动，请在此处注册您的兴趣。我们还将组织免费网络研讨会，为您提供有关该项目的更多详细信息，包括解决方案设计、演示和问答。
