原文链接

NNS 是组织、跟踪和管理 Internet 计算机的节点和子网的控制机制。

作者：Lara Schmid，研究员|DFINITY

通过结合世界各地独立数据中心运行的节点机器的计算能力，互联网计算机托管称为容器的智能合约，允许开发人员在开放的互联网上部署可互操作的软件。

互联网计算机区块链的一个关键特征是网络神经系统（NNS）,一个开放的算法治理系统，负责监督网络和代币经济，使构建 DeFi 和 dapps 成为可能，开放的互联网服务和能够超大规模运行的企业系统。

Internet Computer的ICP实用代币持有者可以将其代币锁定在 neurons 中，参与治理并为决策做出贡献，例如投票决定是否应该将新的节点集合（也称为子网）添加到网络中。通过参与治理，用户可以获得回报。他们也可以使用这些奖励或从交易所获得的 ICP 实用代币来推动其容器的计算。

为何我们需要NNS#

互联网计算机是由节点机器网络运行的分布式协议，它们托管在不同的数据中心。节点通过互联网相互通信，以就互联网计算机的状态达成共识。参与共识的节点集合称为子网。

在这个底层通信和共识协议之上，互联网计算机托管容器，它们是有状态的程序，也可以相互通信。所有容器的状态必须在所有节点上复制。因此，为了让 Internet 计算机无限扩展，网络是由多个子网而不仅仅只是一个组成。

不同的子网可以相互通信，从而使托管在不同子网上的容器也能够相互通信。

让互联网计算机按需扩展，网络必须能够随着时间的推移添加新的子网以增加计算能力。此外，随着时间的推移向子网添加新节点，可以提高子网的稳健性。最终，互联网计算机将大规模运行数百万个节点。

这意味着需要一种机制来组织、跟踪和管理节点和子网。例如，必须决定何时添加或删除子网和节点。此外，Internet 计算机正在推出一个初始功能集，该功能集可能会随着时间的推移而发展。因此，互联网计算机需要能够决定如何以分布式方式发展协议。

NNS 概述#

网络神经系统The Network Nervous System，或“NNS”，是一个代币化的开放治理系统，负责管理互联网计算机。例如，它存储有关哪些节点属于哪个子网的信息。它还决定如何更新此信息。

在高层次上，NNS 是由一组容器实现的。为了做出决定，其中两个容器是关键：

  • 治理容器：这存储了两件事：
  • a.提案，关于如何更改 Internet 计算机的建议。然后可以对这些提案进行投票。
  • b.神经元，决定谁可以参与治理。
  • 注册表容器：这存储了整个 Internet 计算机的配置，然后其他人可以查找这些配置。例如，它存储哪些节点属于哪个子网。

通过 NNS 解析决策#

因此，现在让我们来看看如何提出、投票和实施改变协议的想法。

假设我们有用户 A，他控制着神经元 A，并提交了关于将节点 N 添加到子网 1 的提案。然后，该提案将被添加到治理容器中。其他也控制神经元的用户随后可以对该提案进行投票。如果大多数用户赞成该提议，则该提议将被执行，这意味着注册表容器的内容将被更新以反映节点 N 是子网 1 的一部分。

互联网计算机支持各种不同的提案主题。以下是最初支持的主题的一些非详尽示例：

  • #SubnetManagement 建议：这考虑了拓扑变化。上面关于是否应将节点添加到子网的示例建议属于此类。
  • #ExchangeRate 提案：这涉及有关 ICP 实用代币价值的实时信息，这对于确定必须将多少代币转换为计算周期很重要。 （更多内容如下。）
  • #NodeAdmin 建议：这涉及节点机器的管理。建议的示例可以指定应更新子网中的所有节点。
  • #NetworkEconomics 提案：这涉及网络经济学的管理。例如：应该向节点机器提供者支付什么奖励？

通过 NNS 进行代币管理#

让我们来看看 NNS 上的容器以及它们如何更详细地工作。作为理解神经元概念的先决条件，我们现在来看看用于参与治理系统的 ICP 实用代币。

代币由 NNS 上的另一个容器管理，即分类帐容器，它存储两件事：账户和交易。帐户记录跟踪给定主体 ID（即，在 Internet 计算机上验证用户的身份）拥有多少代币。然后可以将代币从一个帐户发送到另一个帐户，这会记录在分类帐容器的交易中。

ICP 实用代币可用于三种不同的用途：

  • 首先，代币有助于参与治理（下文将详细介绍神经元如何与代币连接）。
  • 其次，参与治理的人和通过操作笔记机提供计算能力的人都获得了代币奖励。
  • 最后，代币用于转换为循环，循环是计算和数据存储容器的燃料。

神经元简介#

神经元包含锁定的代币，这些代币不是流通的，不能自由转移给其他人。

在其他数据中，神经元存储以下信息：

  • 首先，它跟踪有多少代币与给定的神经元相关联，它通过引用分类帐上的帐户来做到这一点。
  • 其次，它指定了一个主体 ID，它标识了一个公钥。
  • 最后，它还指定了一个唯一的神经元 ID。

还有一些与神经元相关的其他参数需要注意。最重要的一项是可以解锁代币的最早日期。只有那些代币被锁定至少六个月的神经元才被允许参与治理。这应该会激励神经元持有者投票，以使他们的代币的价值在未来某个日期最大化。

如果代币的价值是网络成功的粗略代表，这将激励选民为互联网计算机的长期利益投票。

神经元的投票权取决于多种因素，包括锁定在神经元中的代币数量以及直到可以解锁代币的剩余时间。

如何将代币锁定在神经元中#

假设用户 B 在分类帐容器上有一个帐户 (A1)，他想将一百个代币锁定在一个神经元中。为此，用户 B 向 NNS 发送一个命令，指定令牌数量和用户 B 的相应主体 ID。

然后在账本上记录一笔交易，该账本指定一些代币从用户的原始账户 (A1) 发送到新账户 (A2)，新账户 (A2) 也会创建持有锁定代币的新账户 (A2)。在治理容器中创建了一个新神经元，指定用户 B 是控制该神经元的人，并指定锁定令牌的数量由新分类帐帐户 A2 定义。

从外部看，新账户 (A2) 持有锁定的代币或与原始账户 (A1) 有任何关联是不可见的。然而，这个账户实际上是由神经元控制的，这意味着代币不是流通的，用户 B 不能转移代币或将代币转换为循环。

控制神经元使用户能够提交提案并对其进行投票，这将在下面进行解释。

通过 NNS 提交提案#

提议描述了容器中的方法，如果提议被接受，则调用该方法。此外，它还描述了调用该方法的参数。让我们考虑用户 C 建议创建一个新子网的示例，该子网最初由两个节点组成：节点 1 和节点 2。

一旦用户控制了一个神经元，他们就可以通过指定他们的神经元 ID、他们想要提交的提案类型以及提案的参数来提交提案。如果满足要求，则将提案添加到治理容器中。当用户添加新提案时，与给定提案相关的“是”票数会因提案者的投票权而增加。这反映了该提案已经得到了提交它的用户的支持。

为了避免被无用的提案淹没，如果提案被拒绝，提交提案的用户必须支付少量费用。

在提交提案并将其添加到治理容器后，控制神经元的其他用户可以对其进行投票。要投票，用户首先会向治理容器询问待定提案，以了解他们实际可以投票的内容。

假设给定的用户 D 想要拒绝用户 C 刚刚添加的提案。为此，用户 D 将向治理容器发送他们的神经元 ID 和“否定”票。同样，治理罐将检查投票是否来自控制神经元的正确用户，并确认神经元有资格投票。如果满足条件，则治理容器会将用户 D 的投票权添加到“否定”票中。

假设在某一时刻，大多数投票权都支持该提案。该提案将被执行，这意味着将调用所描述的方法。在该示例中，将使用指定的参数调用为注册表容器创建子网的方法。结果是向注册表容器添加了一个新子网，该容器由给定的节点节点 1 和节点 2 组成。

在 NNS 上收集奖励#

参与决策是选民锁定其神经元的一种动力，但他们也因参与网络治理而获得奖励。

每个神经元都会跟踪它通过参与治理积累了多少奖励。为了收集奖励，用户可以向治理容器发送命令以产生新的神经元。结果，将在分类帐上创建一个具有新关联帐户的新神经元，其中包含奖励。

事实上，新的代币被铸造并转移到新账户中，新账户也记录在账本容器中。新神经元只有少许释放延迟时间，这意味着用户只需等待很短的时间即可解锁代币并自由使用。节点机器提供者也通过 NNS 获得奖励。

在 NNS 上委托投票#

用户可能没有时间或知识参与所有网络决策。例如，不熟悉网络拓扑的用户可能不想就是否应将节点添加到子网的提议做出决定。

对于这些情况，互联网计算机促进了流动民主（所谓的追随者）。治理容器跟踪追随者和被追随者神经元的关系。

如果大多数追随者神经元以特定方式投票，则治理容器也会自动记录追随者神经元的相应投票。通过这种方式，追随者神经元仍然可以获得投票奖励，而无需积极参与治理。神经元甚至可以根据特定的提案主题“追随”不同的神经元。

支付周期#

除了治理参与和投票奖励，代币还可以转换为循环，它为互联网计算机上的计算和存储提供燃料。互联网计算机上的每个容器，除了 NNS 上的容器，都使用循环进行计算，并在其中存储了一些循环。虽然代币价格可能会随时间变化，但周期的目标是保持计算价格随时间大致一致。

考虑用户 E，他在 Internet 计算机上运行了一个容器，并且想要充值这个容器的周期，以便它可以执行更多的计算。

还假设这个容器当前有 700 万亿次循环，并且用户 E 想将这个数字增加 200 万亿次。为此，用户将向 NNS 发送一个命令，该命令指定为他们的容器充值的操作。

收到此命令后，会从用户帐户向周期的铸币容器进行交易，该铸币容器是 NNS 上的另一个铸币容器。

作为此交易的结果，周期的铸币容器将燃烧代币，铸币新循环，并将这些新铸币循环发送到用户的铸币容器，这意味着铸币容器余额现在是 900 万亿个循环。
