老徐总结：#

为了便于读者理解，提一下里面的重点：TVL和资本效率。
本文再一次强调资本效率，基本上我们可以视Defi1.0为TVL也就是量，Defi2.0则为 Capital Efficiency 资本效率即质，如果有Defi3.0的话，那么必须基于以上两点去形成一个Z坐标轴，我们可以想象为广度，目前还没有，拭目以待。

目前跨链转移始终需要通过的额外保证层，Mosaic 允许社区成员通过运行具有主动流动性 SDK 的流动性机器人来提供即时 (JIT - Just In Time) 流动性。
这创建了一个动态的跨链流动性市场，Mosaic 自己的再平衡和来自社区的各种机器人合作在竞争奖励的同时进行转移。因此，我们必须为市场和机器人定义理想的功能，以指导奖励分配的设计以及必要的技术栈来实现理想功能。

JIT 机器人在 Mosaic 生态系统中的作用#

Mosaic 是 Composable 的跨链转移可用性层，允许社区成员抢先运行 Mosaic 自己的再平衡，并提供即时流动性，以满足大型跨链转移的需求。如果 Mosaic 自身的再平衡时间过长，流动性机器人可以介入以促进转移并获得更大份额的奖励。

JIT 流动性提供与 Mosaic 的其他功能协同工作，以确保不再有卡住的跨链交易，包含：

  • 单边流动性供给，即社区成员可以只为一种资产提供流动性；
  • 利用机器学习进行流动性预测，这使 Mosaic 能够预测流动性短缺并主动重新平衡其金库；
  • 通过 Connext 或 Hop 等外部桥重新平衡其保险库。 Mosaic 主动流动性模块的高级视图，其中机器人先行运行 Mosaic 的再平衡过程。更多细节在这里。

我们正在构建一个 Active Liquidity Bot SDK，以便社区成员可以安装和运行他们自己的机器人。
当活跃流动性模块上线时，Mosaic 将与其支持的各种网络上提供流动性的大量机器人连接，包括 Ethereum、Polygon、Avalanche、Arbitrum 等。
这使得 Mosaic 成为一个高速、交互式的流动性市场。每个机器人从 Mosaic 提供和提取流动性以最大化他们的回报；它们都对 Mosaic 的流动性做出反应并塑造它。
因此，我们必须了解主动流动性供应的博弈论，以便为所有相关方带来最大化的结果。

在 Composable Finance，我们的目标是激励与 Mosaic 的所有利益相关者（用户、被动流动性提供者、桥和其他 Mosaic 集成的应用程序）一致的机器人行为。下面，我们分享我们对跨链流动性市场的理想特征以及我们如何定义这些机器人的“良好行为”的想法。这些将指导市场设计和奖励分配。

跨链流动性市场的理想功能#

跨链转账的用户体验主要由转账时间和手续费两个因素决定。
反过来，两者都依赖于市场上有足够的流动性并可有效地部署流动性。
虽然 TVL 是目前大多数 DeFi Dapps 的主要 KPI，但我们认为从长远来看，资本效率对于保持可持续收益和锁定 TVL 至关重要。除了财务指标外，还需要去中心化以确保系统的弹性。

资本效率#

DeFi Dapps 不仅与 TVL 有关，还与使用 TVL 的资本效率有关。
如果通过 Mosaic 自身的再平衡和积极的流动性供应，1000 万美元足以避免金库耗尽或高额转账费用，则无需将 1 亿美元锁定在流动性金库中。
更高的效率意味着奖励可以与更小的资本基础共享，这意味着所涉及的流动性提供者的 APY 更高。

我们希望促进主动机器人准确响应网络需求；准确的反应有两个方面：提供流动性的数量和提供流动性的时间。

  • 在流动性方面，对超过给定阈值的转账做出反应的机器人应该比那些不针对任何转账的机器人获得更多的奖励；
  • 就时间而言，资本效率要求“及时”而不是“随时”提供流动性。
    有关更多详细信息，请参阅“JIT 机器人的理想功能”部分（下文）。

去中心化的权力下放#

在稳定状态下，美盛的流动性供应不应依赖于任何流动性供应方。如果一个保险库需要 2000 万美元的流动性，那么让五方各提供 400 万美元比让两方提供 1000 万美元更可取；去中心化将中心点故障的风险降到最低，也增加了竞争和激励流动性提供方做出响应。最好有多个参与者支持金库并与社区广泛分享奖励，而不是让一个参与者控制金库和囤积奖励。

Mosaic 的跨链流动性保险库设置通过设计支持去中心化。由于一个金库是一条链上的一个代币，假设每条链有五个代币，支持十条链，那么大约有 50 个金库，因此一方很难影响整体上 Mosaic 的流动性。

另一个创新功能是 JIT 流动性机器人可以向 Mosaic 支持的任何链提供和提取流动性；这允许机器人响应不同链上动态的需求/供应，而不是囤积在给定的保险库上。
对金库的需求和供应会随着时间的推移而变化，以响应网络活动（新产品发布、流动性热潮等）以及流入和流出金库的主动和被动流动性的数量。对于大型资金提供者而言，将资金划分为不同的金库提供流动性很可能比将所有资金放在一个金库中提供更高的收益；这意味着积极的流动性提供者在经济上受到去中心化激励。

JIT 机器人的理想功能#

我们希望激励活跃且关注 Mosaic 流动性动态的机器人，这可以大致分为三个标准：

即时提供流动性#

我们希望激励在中继者转移前一刻提供流动性，这样可以更好地了解金库上的流动性水平。在几个区块的范围内可能会发生很多事情。如果机器人过早提供流动性，则在中继器对转移采取行动时，金库流动性可能会发生很大变化。

提供足够的流动性#

好的机器人在提供活跃的流动性时会很有效。好的机器人应该以受过教育的方式参与系统，表明他们正在关注保险库流动性需求。理想情况下，机器人将看到 X 代币的转移，并提供 X 代币 + 5% 的 X 代币，以考虑到供应流动性和中继器根据转移需求而变化的流动性水平。当只需要 2% 来促进转移时，他们不应该只是盲目地提供 50% 的金库流动性。

即时提取流动性#

一个好的机器人将确保其资金高效，并且其目标是将流动性保持在“足够”的水平。
这部分实现来自于通过激励机器人在最小活跃流动性锁定时间后立即尝试退出的机制，即机器人在退出之前需要等待的最短时间。这种行为还有助于保持更高的货币流通速度，从而使金库更有效率。

获得跨链可提取价值的好处#

MEV 策略已经获得了巨大的利润；
Mosaic 向任何链提供和提取流动性的能力开辟了一个新的领域，即“跨链可提取价值” （可理解为商品生产和交换过程中的剩余价值–风车注）。
MEV与跨链可提取价值的策略有不少重叠；因此，我们预计 MEV 探索者可以在短时间内开发出有利可图的跨链策略。
例如，最常见的 MEV 形式“三明治”通过提前运行大宗挂单并从大宗订单的价格影响中获利，这类似于 JIT 流动性机器人在 Mosaic 上的工作方式——它们为目标链提供流动性预计会有大量的转移请求。
然而，重要的是，就 Mosaic 而言，领先者并没有为最终用户的利润而竞争，而是在促进跨链转移，否则可能需要更长的时间才能完成。
“套利”是另一种常见的 MEV 策略，涉及在一个地方购买资产并在另一个地方出售以利用价格差异，它将在 Mosaic 上找到新的应用——JIT 流动性机器人可以在以太坊链上提供 USDC 并在 Arbitrum 上提取它利用相同资产在不同链上的收益率差异。

在不同链上操作的灵活性也开辟了超越 MEV 常见策略的新策略。
我们积极的流动性领导者之前曾对这些策略进行过调侃，但回顾一下，有以下灵活发挥的地方：

  • 向同一网络提供和提取流动性（策略 1）
  • 使用主动流动性 SDK 中的 API 检查提现的最低网络费用，然后提现到费用最低的地方（策略 2）
  • 设置网络退出并等待费用低于某个阈值时，这允许您在赚取费用的同时转移到另一个链（策略3）
  • 在不同的网络上设置两个机器人并循环流动性供应/取款策略。在网络 A 上提供流动性，向 B 提款，使用流动性在 A 上提供流动性，然后在 A 上提款，然后重复（策略 4）
  • 将部分流动性转换为本地 gas 代币以降低网络费用。 （策略 5）

跨链可提取价值为 MEV 探索者开辟了令人兴奋的机会。
我们将通过活跃的流动性 SDK 和实践指南促进针对 MEV 社区并促进跨链可提取价值的入门。
目前正在开发的主动流动性 SDK 允许用户在 Mosaic 支持的任意多条链上设置任意数量的 JIT 机器人实例；它将随附指南和代码示例。
我们还将分享操作指南、视频剪辑和主持 AMA，让社区熟悉跨链 LP 机会。

总结#

主动流动性 SDK 将允许社区成员运行机器人并提供主动流动性。
主动流动性将与 Mosaic 的被动流动性合作，确保转账通过并争夺奖励，创建一个跨链流动性市场。
在本文中，我们分享了市场和机器人行为的设计原则。
我们的目标是创建一个专注于资本效率和去中心化的市场；
我们希望确保流动性提供者能够最大化他们的回报，并且减少对大型资本提供者的依赖，而不仅仅是高 TVL。
对于机器人，我们的目标是为那些在正确的时间和数量提供和提取流动性的人提供更高的奖励；
我们还将制定充分利用 Mosaic 跨链流动性供应机会的策略。
