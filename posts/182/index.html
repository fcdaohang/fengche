因为我的阅读障碍，所以做了这些内容

首先是全局一览：#

Layer 0#

Layer 0 是整个区块链结构的最底层，这是互联网、硬件和连接集中的地方，它们允许像比特币这样的 Layer 1 顺利运行。Layer 0 允许这些发生：

允许区块链之间的交互（互操作）#

  • 一个很好的例子是 Cosmos，它通过其“Tendermint IBC”（区块链间通信协议）创建了一个可互操作的区块链生态系统。
  • 对于开发人员来说意义巨大。如果一个 Dapp 可以在一个区块链上运行，也就可以在其他区块链上运行，只要它们是使用相同的 Layer 0 构建的。无需投入更多时间和资源在另一个链上构建相同的应用程序。

更快更低成本的交易#

使用 IBC，PoS 共识可以跨多个链实现，导致最终确定时间几乎瞬间发生（最终确定 = 当一个块被批准时，不能回滚，并且被认为是不可逆的）。结果是跨链交易所的交易更快、更便宜。

便于开发人员的基础设施#

不需要从头开始构建他们之前开发的区块链。许多功能都是预先构建的，可以通过 Layer 0 立即实施。

Layer 1#

Layer 1 是在专属区块链上处理和完成交易的区块链平台（比特币和以太坊）。它们之间诸如共识（PoW、PoS）以及区块生成时间和争议解决之类的所有技术细节存有差异。

  • Layer 1 区块链最重要的三个方面也是其需要克服的三难困境，即：去中心化、安全性和可扩展性。目前仍然没有一个 Layer 1 区块链能够同时解决这三个问题。 （为了解决这个三难困境问题，由此引入了后来的 Layer 2 甚至 Layer 3 – 风车注）

Layer 2#

Layer 2 是与 Layer 1 结合使用的第三方集成，以提高可扩展性和性能（每秒事务数/系统吞吐量/TPS）。

  • 当你听到零知识证明（zk rollups）、侧链或任何与加快事务吞吐量有关的事情时，很可能是 Layer 2。

Layer 3#

Layer 3 目前定义是应用层（Dapps）。这是我们作为消费者实际与之交互的 UI 界面。
